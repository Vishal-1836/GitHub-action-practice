Problem 2 — Pull Request Validation Workflow
Concept Covered

pull_request trigger
Branch protection style checks
PR validation pipeline

Problem Statement
Create a workflow that runs only when a Pull Request is created against the main branch.
The workflow should:

Checkout the repository
Run a basic validation command
Print a success message if validation passes
Fail the workflow if a required file like README.md is missing

Expected Learning
You should understand:

How PR-based workflows work
Difference between push and pull_request
How GitHub Actions can protect branches before merge


**Problem 2 — Environment Variables & GitHub Contexts**

    Concept Covered: env at workflow/job/step level, github.*, runner.*, secrets.* contexts
    
    Scenario:
    
    Your team wants to standardize environment variables across pipelines.
    
    Tasks:
    
    Define APP_NAME=payflow at workflow level
    Define ENVIRONMENT=dev at job level
    Define REGION=centralindia at step level
    Print all 3 in a single step
    Print these GitHub contexts:
      
      ${{ github.repository }}
    ${{ github.workflow }}
    ${{ runner.os }}
    ${{ runner.arch }}

Success Criteria: Understand variable scope precedence (step > job > workflow).

        name: practice-problem-2
        env:
          APP_NAME: ram-practice-project
        on:
          pull_request:
            branches: ["main"]

            env:
              ENVIRONMENT: ram
            runs-on: ubuntu-latest
            steps:
              - name: Checkout
                uses: actions/checkout@v7.0.0

                env:
                  REGION: centralindia

                  echo "The workflow level varible : ${{env.APP_NAME}}"
                  echo "The job level varible : ${{env.ENVIRONMENT}}"
                  echo "The step level varible : ${{env.REGION}}"
                  echo "The repo name : ${{ github.repository }}"
                  echo "The workflow name :${{ github.workflow }}"
                  echo "The runner os :${{ runner.os }}"
