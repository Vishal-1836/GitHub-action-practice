Problem 2 — Pull Request Validation Workflow
Concept Covered

pull_request trigger
Branch protection style checks
PR validation pipeline

Problem Statement
Create a workflow that runs only when a Pull Request is created against the main branch.
The workflow should:

Checkout the repository
Run a basic validation command
Print a success message if validation passes
Fail the workflow if a required file like README.md is missing

Expected Learning
You should understand:

How PR-based workflows work
Difference between push and pull_request
How GitHub Actions can protect branches before merge
