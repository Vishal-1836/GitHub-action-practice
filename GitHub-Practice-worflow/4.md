**Problem 4 — Multi-Job Pipeline with Dependencies**

Concept Covered: needs, job dependencies, sequential & parallel execution, if conditions

Scenario:

You're building a pipeline for a Node.js microservice with proper stage gating.

Tasks:

Create a pipeline with this flow:

        ┌──► test ──┐
lint ───┤           ├──► build ──► deploy
        └──► scan ──┘


lint — runs ESLint check

test and scan — run in parallel after lint

build — runs only if both test AND scan succeed

deploy — runs only on push to main (not on PR)

If lint fails, nothing else should run

Success Criteria: Pipeline blocks at first failure; deploy never runs on PRs.
